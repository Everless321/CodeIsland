# MCP Server: SSE → Streamable HTTP

**日期**: 2026-04-16
**类型**: 重构

## 任务描述

把 Charon MCP 服务器从旧版 SSE 传输协议 (2024-11-05) 升级到 Streamable HTTP (2025-03-26)。

## 待办

- [x] 查 Claude Code 配置格式（`type: "http"`）
- [x] 重写 `MCPServer.swift`：单端点 POST /mcp，MCP-Session-Id header 管理会话
- [x] 更新 `ConfigInstaller.swift`：`type: sse → http`，`url: /sse → /mcp`
- [x] 编译验证
- [x] 提交推送 + 打包重装

## 执行结果

### MCPServer.swift

- 协议版本 `2024-11-05` → `2025-03-26`
- 端点 `/sse` + `/messages?sessionId=<id>` → **单端点 `/mcp`**
- 会话标识从 URL 查询参数 → `Mcp-Session-Id` HTTP header
- 删除 SSESession 长连接、keepalive 定时器、SSE 断线监控
- POST 请求直接在 HTTP 响应中返回 `application/json`（不再 202 + SSE push）
- 审批流程通过 HTTP 长连接阻塞等待（localhost 无代理超时问题）
- 支持 `DELETE /mcp` 显式终止会话
- `GET /mcp` 返回 405（暂不使用服务端主动推送）

### ConfigInstaller.swift

- `mcpServers.charon.type`: `sse` → `http`
- `mcpServers.charon.url`: `http://127.0.0.1:9800/sse` → `http://127.0.0.1:9800/mcp`

### 代码简化

- 删除约 80 行 SSE 相关代码（SSESession 类、keepAliveTimer、monitorSSEDisconnect、handleSSE 等）
- `list_servers / save_server` 变成纯同步请求-响应
- `request_server_info / delete_server`（需审批）仍能阻塞等待，HTTP 长连接替代 SSE 推送

## 注意事项

- 端口保持 `127.0.0.1:9800` 不变
- 现有已接入的 CLI 下次启动 CodeIsland 会自动重写配置
- 已运行的 Claude 实例需要重启才能连上新端点
